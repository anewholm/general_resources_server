00023    /* TXml (Transactional XML) based update
00024     * DOM (updates only) specification: http://www.w3.org/TR/DOM-Level-2-Core/core.html#i-Document
00025     * Document:
00026     *   Type create<Type>(in DOMString tagName);
00027     * Nodes:
00028     *   Node insertBefore(in Node newChild, in Node refChild);
00029     *   Node replaceChild(in Node newChild, in Node oldChild);
00030     *   Node removeChild(in Node oldChild);
00031     *   Node appendChild(in Node newChild);
00032     *   Node cloneNode(in boolean deep);
00033     *   (property) nodeValue(in DOMString value);
00034     * Attribute Named Node Maps:
00035     *   Node getNamedItem(in DOMString name);
00036     *   Node setNamedItem(in Node arg);
00037     *   Node removeNamedItem(in DOMString name);
00038     *   Node setNamedItemNS(in Node arg);
00039     *   // Introduced in DOM Level 2:
00040     *   Node removeNamedItemNS(in DOMString namespaceURI, in DOMString localName);
00041     *
00042     * BUT: We want a one-command, no persistent variables, with transforms based update capacity
00043     * XmlBaseDoc can understand it and supervise it with Repository without LibXmlBaseDoc needing to help
00044     * Request can communicate directly with the chosen IXmlBaseDoc with TXml objects through pure virtual ITXmlProcessor interface below
00045     *  or directly as it wishes
00046     *  using a setteable IXmlBaseNode *pBaseNode for following xmltransactions rather than an entire Database object + manager
00047     *  entire Database object + manager therefore moves in to the Request object
00048     * This is also what is used in the source XML now as well
00049     * <xmltransaction:move-child
00050     *   document-name="[name]"
00051     *   base-node or database-name="[unique-xpath, often id('db_test')]"
00052     *   [from="[unique-xpath]" to="[unique-xpath]" ...]
00053     *   [created="[UNIX timestamp]"]
00054     *   [xmlsecurity:username="[user]" xmlsecurity:password_hash="[pass]"]
00055     *   [record="false"]
00056     *   [immediacy="high"]
00057     *   [applied="yes"]
00058     * >
00059     *   [the single optional xml parameter]
00060     * </xmltransaction:move-child>
00061     *
00062     * move and copy, combined with transform give real power over the pre-defined XML to be added
00063     * outerXML allow arbitrary, user generated, XML in the system
00064     * @destination = appendChild, @before = insertBefore
00065     *
00066     * all actions require a @select node and can use a @base node
00067     */
